AI 助手代理设置指南

当您通过 GitHub Pages 访问本网站时，浏览器的安全限制会阻止直接连接到本地运行的 Ollama 服务。为了解决这个问题，我们提供了代理设置选项。

什么是代理服务？

代理服务充当中间人，允许从 HTTPS 网站（如 GitHub Pages）连接到 HTTP 服务（如本地 Ollama）。

如何使用代理连接到本地 Ollama

  1. 确保您的本地 Ollama 服务正在运行
  2. 在 AI 助手对话框中，您会看到”代理设置”选项
  3. 从下拉菜单中选择一个代理服务
  4. 点击”保存并重新连接”按钮

可用的代理选项

  • 默认代理：使用 CORS Anywhere 服务
  • 备用代理：如果默认代理不可用，可以尝试备用选项
  • 自定义代理：您可以输入自己的代理服务 URL

自托管代理

如果您希望更好的隐私保护，可以自托管代理服务：

  1. 克隆 CORS Anywhere 仓库
  2. 按照说明在本地或您自己的服务器上部署
  3. 在 AI 助手设置中使用您的自托管代理 URL

故障排除

如果连接失败：

  1. 确认 Ollama 服务正在本地运行（http://localhost:11434/api/tags 可访问）
  2. 尝试不同的代理服务
  3. 检查代理服务是否有使用限制或需要授权

隐私说明

使用第三方代理服务时，您的请求和 Ollama 的响应将通过该服务传输。虽然内容仍然在您的设备和本地 Ollama 之间传输，但代理服务理论上可以看到这些数据。

如果您对隐私有严格要求，建议使用自托管代理或在本地环境中使用本网站。
